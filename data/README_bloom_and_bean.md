# Bloom & Bean — synthetic test dataset

A fictional **online coffee roastery & equipment store**. Generated by
[`generate_business_data.py`](../generate_business_data.py) to exercise every
agent in Smart Analyst. Deterministic (seed = 7), so re-running reproduces it.

## Files
| File | Rows | Use |
|------|------|-----|
| `bloom_and_bean_sales.csv` | ~5,700 | **Dirty** — upload this to **Data Cleaning** |
| `bloom_and_bean_sales_clean.csv` | ~5,640 | Clean reference (diffing / direct engine tests) |

24 months of daily orders (2023-06-01 → 2025-05-31), 420 repeat customers,
~3,300 orders, ~$640K revenue.

## Columns
| Column | Type | Notes |
|--------|------|-------|
| `order_id` | id | groups multi-line orders (1–3 lines each) |
| `order_date`, `ship_date` | date | a few malformed values in the dirty file |
| `customer_id` | id | stable across orders → enables RFM |
| `customer_name`, `email` | text | |
| `product`, `category` | category | Coffee Beans, Equipment, Accessories, Gifts, Subscription |
| `quantity` | numeric | some string/outlier junk in dirty file |
| `unit_price` | numeric | per-unit |
| `discount_pct` | numeric | 0–20% |
| `total_price` | numeric | **line revenue** = qty × price × (1−disc) |
| `cost`, `profit` | numeric | enables margin / profitability analysis |
| `region`, `country` | category | geo dimensions |
| `marketing_channel` | category | Organic/Paid/Social/Email/Referral/Direct |
| `payment_method` | category | |

## What each agent should find
- **Cleaning** — missing values + placeholders (`Unknown`, `N/A`, `-`, `ERROR`),
  type mismatches in `quantity`/`unit_price`, price/qty outliers, ~55 duplicate
  rows, malformed dates, inconsistent region casing. Plus the arithmetic
  relationship `profit = total_price − cost`.
- **Analytics / KPIs** — revenue, AOV (~$194), 420 customers, ~74% repeat rate,
  category & product breakdowns, monthly growth.
- **Time-series** — clear **upward trend**, weekly seasonality, **Q4 holiday +
  Black-Friday spikes** (anomalies/spikes), and a **March supply-outage dip**
  (structural break).
- **Forecasting** — daily series; `Auto` back-tests models and should beat naive
  on `total_price` / `profit` / `quantity`.
- **Marketing / RFM** — full segment spread (Champions, At Risk, Loyal,
  Hibernating, Lost, …) and channel/region/country/payment performance.
